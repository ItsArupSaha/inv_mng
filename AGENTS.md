# Interaction Constraints
- DO NOT open the Scratchpad, Playground, or separate temp tabs.
- All tasks, changes, and histories must remain directly in the active project view.

# Coding Guidelines & Modular Design
- **Prioritize Modularity**: Avoid creating large, monolithic component files. If a component or view grows past 300 lines, proactively extract sub-components, custom hooks, or business logic into separate, dedicated files under a logical folder structure.
- **Refactoring Over Appending**: When adding features, refactor existing code to accommodate the changes elegantly instead of simply appending block code. Maintain a clean separation of concerns.
- **Follow Clean Code Patterns**: Write clean, professional code following standard design patterns. Use descriptive naming, extract helpers, and utilize typescript types/interfaces effectively to promote reuse and readability.
